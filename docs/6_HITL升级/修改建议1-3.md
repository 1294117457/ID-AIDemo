> 修正了 1-2.md 中对 student 表的误判：实际上没有独立的 student 表，
> 学生信息全部存在 `users` 表中。

## 一、DB 结构关键确认

### users 表（已整合学生信息）

```sql
CREATE TABLE `users` (
  `id`          int          -- user_id，JWT payload 中存的就是这个
  `username`    varchar(255) -- 如 33120202201909@stu.xmu.edu.cn
  `full_name`   varchar(100) -- 姓名，如"周晨辉"
  `grade`       year         -- 入学年份，如 2003
  `major`       varchar(100) -- 专业，如"软件工程"
  ...
)
```

- **无独立 student 表**，学号直接从 `username` 提取：`33120202201909@stu.xmu.edu.cn` → `33120202201909`
- Java `AuthInterceptor` 解析 JWT 后已在 request 中设置 `userId = users.id`

### score_applications 表（提交申请时写入的字段）

```sql
CREATE TABLE `score_applications` (
  `user_id`        int          -- FK → users.id
  `student_id`     varchar(50)  -- 从 username 提取的学号
  `student_name`   varchar(100) -- users.full_name
  `major`          varchar(100) -- users.major
  `enrollment_year` int         -- users.grade
  `template_name`  varchar(100) -- 模板名称
  `score_type`     tinyint      -- 模板的 score_type (0/1/2)
  `apply_score`    decimal(5,2) -- 申请分数
  `rule_id`        int          -- 匹配到的规则 ID
  `review_count`   int          -- 需要几个审核员（来自模板）
  `remark`         text
  ...
)
```

### application_proofs 表（每条证明材料）

```sql
CREATE TABLE `application_proofs` (
  `application_id` int
  `proof_file_id`  bigint  -- FK → file_metadata.id（前端上传后得到的 fileId）
  `proof_value`    decimal(10,2)
  `review_count`   int
  `remark`         varchar(500)
)
```

## 二、整体数据流（修正版）

```
前端
  │  Authorization: Bearer <token>
  │  POST /api/agent/stream (message + file + sessionId)
  ▼
Java AIAgentController
  │  userId = httpRequest.getAttribute("userId")   ← 已有
  │  查 users 表 → full_name, grade, major, username
  │  派生 studentId = username.split("@")[0]
  │  拼装 userInfo JSON → 随 multipart 发给 Node Agent
  ▼
Node Agent
  │  解析 userInfo → 存入 MainState.userInfo
  │  LangGraph: classify → applyGraph
  │    ├─ fetchPolicy
  │    ├─ analyzeAndMatch（使用 state.templates 匹配）
  │    ├─ summarize（输出匹配结果）
  │    └─ confirm（interrupt：展示结果 + 要求上传文件确认）
  │  SSE 推送 interrupt 事件（含 suggestions 列表）
  ▼
前端
  │  展示匹配结果 + 文件上传区 + 确认按钮
  │  用户上传证明文件 → 调现有文件上传接口 → 得到 fileId 列表
  │  用户确认 → resume（JSON: {confirm, proofFileIds, proofValues}）
  ▼
Node Agent（恢复）
  │  submitNode:
  │    - 从 state.userInfo 取用户信息
  │    - 从 state.templates 查完整模板（scoreType, reviewCount 等）
  │    - 拼装提交数据
  │    - POST /internal/agent/submit → Java
  │  SSE 推送 result（"申请已提交成功，编号 xxx"）
  ▼
Java InternalAgentController
  │  校验 X-Internal-Service-Key
  │  写 score_applications + application_proofs
  │  返回 applicationId
```

### 3.1 Java：AIAgentController 查 users 表构建 userInfo

`AuthInterceptor` 解析 JWT 后已在 request attribute 中放了 `userId`，直接取用。
在 `AIAgentController` 注入 `UserMapper`（已有的），查一次 `users` 表即可：

```java
// AIAgentController.java 中新增
@Autowired  // 或通过构造注入
private UserMapper userMapper;

private String buildUserInfoJson(HttpServletRequest httpRequest) {
    Integer userId = (Integer) httpRequest.getAttribute("userId");
    if (userId == null) return "{}";

    // 查 users 表（已有的 UserPO 或 User 实体）
    UserPO user = userMapper.findById(userId);
    if (user == null) return "{}";

    // 从 username 提取 studentId
    String username = user.getUsername();  // 33120202201909@stu.xmu.edu.cn
    String studentId = username.contains("@") ? username.split("@")[0] : username;

    Map<String, Object> info = new HashMap<>();
    info.put("userId", userId);
    info.put("studentId", studentId);
    info.put("studentName", user.getFullName() != null ? user.getFullName() : "");
    info.put("major", user.getMajor() != null ? user.getMajor() : "");
    info.put("enrollmentYear", user.getGrade() != null ? user.getGrade() : 0);
    return JSON.toJSONString(info);
}
```

在 `stream` 和 `chat` 方法中调用：

```java

### 3.3 Java：新增 InternalAgentController

新建文件，接收 Agent 的回调提交请求：

```java
@RestController
@RequestMapping("/internal/agent")
public class InternalAgentController {

    @Value("${internal.service-key}")
    private String serviceKey;

    @Autowired
    private ScoreApplicationService scoreApplicationService;  // 已有

    @PostMapping("/submit")
    public ResponseEntity<Map<String, Object>> submit(
            @RequestHeader(value = "X-Internal-Service-Key", required = false) String key,
            @RequestBody AgentSubmitDto dto) {

        if (!serviceKey.equals(key)) {
            return ResponseEntity.status(403)
                .body(Map.of("code", 403, "msg", "forbidden"));
        }

        try {
            Long applicationId = scoreApplicationService.submitFromAgent(dto);
            return ResponseEntity.ok(Map.of("code", 200,
                "data", Map.of("applicationId", applicationId)));
        } catch (Exception e) {
            return ResponseEntity.status(500)
                .body(Map.of("code", 500, "msg", e.getMessage()));
        }
    }
}
```

**AgentSubmitDto.java**（对应 score_applications + application_proofs 所需字段）：

```java
@Data

    private String templateName;     // score_templates.template_name

    private Double applyScore;       // 匹配的预计分数

### 3.7 Node Agent：applyGraph.ts 新增 confirmNode 和 submitNode

#### confirmNode（interrupt 等待用户确认 + 上传文件）

```typescript
import { interrupt } from '@langchain/langgraph'
import { HumanMessage, AIMessage } from '@langchain/core/messages'

async function confirmNode(state: ApplyStateType): Promise<Partial<ApplyStateType>> {
  const suggestions = state.checkResults
    .map(r => { try { return JSON.parse(r) } catch { return null } })
    .filter(Boolean)

  // 无匹配时跳过 interrupt，直接结束
  if (suggestions.length === 0) return {}

  const summary = suggestions.map((s: any, i: number) =>
    `${i + 1}. **${s.templateName}** / ${s.ruleName}（预计 ${s.estimatedScore} 分）\n   理由：${s.reason}`
  ).join('\n')

  const question = [
    `已为您匹配到以下加分项：\n\n${summary}`,
    '',
    '请上传对应证明材料（使用页面上传功能获取文件 ID），然后回复以下 JSON 确认提交：',
    '```',
    '{"confirm":true,"proofFileIds":[文件ID1,文件ID2],"proofValues":[对应分值1,对应分值2]}',
    '```',
    '如需取消，回复：cancel',
  ].join('\n')

  // interrupt 暂停图，把 question 推送给前端
  const userAnswer = interrupt(question)

  return {
    messages: [
      new AIMessage(question),
      new HumanMessage(String(userAnswer)),
    ],
  }
}
```

#### submitNode（解析回复 + HTTP 回调 Java）

```typescript
const JAVA_URL = process.env.JAVA_BACKEND_URL ?? 'http://localhost:8080'
const SERVICE_KEY = process.env.INTERNAL_SERVICE_KEY ?? ''

async function submitNode(state: ApplyStateType): Promise<Partial<ApplyStateType>> {
  // 取 confirmNode interrupt 之后用户的回复
  const lastHuman = state.messages
    .filter(m => m instanceof HumanMessage)
    .at(-1)
  const answer = String(lastHuman?.content ?? '').trim()

  if (answer.toLowerCase() === 'cancel') {
    return { messages: [new AIMessage('已取消申请，您可以随时重新开始。')] }
  }

  // ── 解析用户回复的 JSON ──
  let proofFileIds: number[] = []
  let proofValues: number[] = []
  try {
    // 提取 JSON（用户可能夹杂在文本中）
    const match = answer.match(/\{[\s\S]*\}/)
    if (!match) throw new Error('未找到 JSON')
    const parsed = JSON.parse(match[0])
    if (!parsed.confirm) throw new Error('未确认')
    proofFileIds = Array.isArray(parsed.proofFileIds) ? parsed.proofFileIds : []
    proofValues  = Array.isArray(parsed.proofValues)  ? parsed.proofValues  : []
  } catch {
    return { messages: [new AIMessage(
      '格式不正确，请重新发送确认 JSON，或回复 cancel 取消。\n示例：`{"confirm":true,"proofFileIds":[1,2],"proofValues":[2.0,1.0]}`'
    )] }
  }

  if (proofFileIds.length === 0) {
    return { messages: [new AIMessage('请至少上传一个证明材料（proofFileIds 不能为空）。')] }
  }

  // ── 检查必要数据 ──
  if (!state.userInfo) {
    return { messages: [new AIMessage('获取用户信息失败，请重新登录后再申请。')] }
  }

  // 取第一个（最优）匹配结果
  const suggestion = state.checkResults
    .map(r => { try { return JSON.parse(r) } catch { return null } })
    .filter(Boolean)[0] as any

  if (!suggestion) {
    return { messages: [new AIMessage('申请数据异常，请重新上传证明材料。')] }
  }

  // 从 state.templates 查找完整模板（需要 scoreType, reviewCount, templateType）
  const fullTemplate = state.templates.find(t => t.id === suggestion.templateId)

  // ── 拼装提交数据（对应 AgentSubmitDto）──
  const submitBody = {
    userId:         state.userInfo.userId,
    studentId:      state.userInfo.studentId,
    studentName:    state.userInfo.studentName,
    major:          state.userInfo.major,
    enrollmentYear: state.userInfo.enrollmentYear,

    templateName: suggestion.templateName,
    templateType: fullTemplate?.templateType ?? 'CONDITION',
    scoreType:    fullTemplate?.scoreType ?? 0,
    applyScore:   suggestion.estimatedScore,
    ruleId:       suggestion.ruleId,
    reviewCount:  fullTemplate?.reviewCount ?? 1,
    remark:       `AI 智能匹配 - ${suggestion.reason ?? ''}`,

    proofItems: proofFileIds.map((id: number, i: number) => ({
      proofFileId: id,
      proofValue:  proofValues[i] ?? 0,
      remark:      '',
    })),
  }

  // ── 回调 Java 内部接口 ──
  try {
    const resp = await fetch(`${JAVA_URL}/internal/agent/submit`, {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
        'X-Internal-Service-Key': SERVICE_KEY,
      },
      body: JSON.stringify(submitBody),
    })
    const data = await resp.json() as any
    if (data.code === 200) {
      return { messages: [new AIMessage(
        `✅ 申请已提交成功！申请编号：${data.data?.applicationId}，请等待审核。`
      )] }
    }
    return { messages: [new AIMessage(`提交失败：${data.msg}，请稍后重试或手动提交。`)] }
  } catch (e) {
    console.error('[submitNode] 回调 Java 失败:', e)
    return { messages: [new AIMessage('网络异常，提交失败，请稍后重试或手动提交。')] }
  }
}
```

#### confirmRoute（路由决策）

```typescript
function confirmRoute(state: ApplyStateType): 'submit' | 'end' {
  // summarize 之后：无匹配则直接结束，有匹配则进 confirm
  const suggestions = state.checkResults
    .map(r => { try { return JSON.parse(r) } catch { return null } })
    .filter(Boolean)
  return suggestions.length > 0 ? 'submit' : 'end'
}
```

> 注意：`confirmNode` 内部有 interrupt，interrupt resume 后继续走 `submitNode`，
> 所以图里 `confirm → submit` 是直连的（interrupt 是 node 内部逻辑）。

#### 更新后的图结构

```typescript
export const applySubgraph = new StateGraph(ApplyState)
  .addNode('fetchPolicy',     fetchPolicyNode)
  .addNode('analyzeAndMatch', analyzeAndMatchNode)
  .addNode('summarize',       summarizeNode)
  .addNode('confirm',         confirmNode)    // ← 新增（内部有 interrupt）
  .addNode('submit',          submitNode)     // ← 新增

  .addEdge(START, 'fetchPolicy')
  .addEdge('fetchPolicy', 'analyzeAndMatch')
  .addEdge('analyzeAndMatch', 'summarize')
  .addConditionalEdges('summarize', confirmRoute, {  // ← 改：原来是 summarize → END
    submit: 'confirm',
    end:    END,
  })
  .addEdge('confirm', 'submit')   // ← 新增
  .addEdge('submit',  END)        // ← 新增
  .compile()
```

### 3.8 Node Agent：agentService.ts interrupt 事件扩展

当前 interrupt 事件格式：`{ type: 'interrupt', data: { question: '...' } }`

建议扩展，把 `suggestions` 单独输出，方便前端渲染列表而不用解析文本：

```typescript
// streamAgent 中，检测到 interrupt 后
const interruptResult = await checkInterrupt(config)
if (interruptResult) {
  const state = (await app.getState(config)).values as any
  const suggestions = (state.checkResults ?? [])
    .map((r: string) => { try { return JSON.parse(r) } catch { return null } })
    .filter(Boolean)

  yield {
    type: 'interrupt',
    data: {
      question: interruptResult.question,
      suggestions,           // ← 新增，前端直接用来渲染列表
      requireFiles: true,    // ← 新增，告诉前端需要展示文件上传区
    }
  }
  return
}

## 七、注意事项

**① ScoreTemplate.reviewCount**：submitNode 里用 `fullTemplate?.reviewCount ?? 1`，如果 Java 未传则默认 1，不会报错。

**② confirmNode 无匹配时不触发 interrupt**：`confirmRoute` 在 `summarize` 之后判断，suggestions 为空直接走 `end`，整个流程结束，不会弹空的确认框。

**③ 用户未填写 full_name/major 时**：`studentName` 和 `major` 可能是空字符串（见数据中用户 25、38），`submitNode` 中直接传过去即可，Java 写库允许 `null`/空字符串。

**④ Node.js fetch**：Node 18+ 有原生 fetch。如果你的环境是 Node 18 以下，需要 `npm install node-fetch`，并在 `applyGraph.ts` 顶部 `import fetch from 'node-fetch'`。

**⑤ SecurityConfig 白名单**：只开放 `/internal/**`，但建议加上 IP 白名单（限制只允许 localhost），防止外网直接调用。
