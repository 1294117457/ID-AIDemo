# 测试文档 1 — HITL 升级 + userInfo 透传 完整测试

> 本次修改以 `修改建议1-3.md` 为基准实施，涵盖 Java 后端 / Node Agent / 前端三层，测试目标为验证以下功能：
> 1. JWT userId 正确透传到 Agent state
> 2. 悬浮聊天气泡（咨询模式）文件上传入口已移除
> 3. 加分申请页改为对话式 AI（含 HITL：interrupt → 文件上传 → 确认 → 内部提交）

## 一、修改内容说明

### 1.1 Java 后端（idbackend）

| 文件 | 修改点 |
|------|--------|
| `AIAgentController.java` | 新增 `buildUserInfoJson()` 方法，从 `request.getAttribute("userId")` 取 userId，查 `users` 表获得 `fullName/major/grade`，序列化成 JSON；在 `stream` / `chat` 方法中调用并传给 `AIAgentService` |
| `AIAgentService.java` | `agentStream` / `agentChat` 新增 `String userInfo` 参数，将其作为 multipart form-data 字段 `userInfo` 转发给 Node Agent |
| `AgentSubmitDto.java` *(新建)* | Agent 内部回调时的请求体 DTO，字段：`userId / studentId / studentName / major / enrollmentYear / templateName / templateType / scoreType / applyScore / ruleId / reviewCount / remark / proofItems[]` |
| `InternalAgentController.java` *(新建)* | `POST /internal/agent/submit`，添加了 `@PublicAccess`（绕过 JWT），通过 `X-Internal-Service-Key` 鉴权，调用 `ApplicationService.submitFromAgent` |
| `ApplicationService.java` | 新增 `submitFromAgent(AgentSubmitDto)` 方法，直接以 `dto.userId` 写库，无需 JWT |

### 1.2 Node Agent（ID-AIDemo）

| 文件 | 修改点 |
|------|--------|
| `state.ts` | 内联 `TemplateRule / ScoreTemplate` 类型（原 `scoreTemplate.ts` 重新导出）；新增 `UserInfo` 接口和 `userInfo` 注解字段 |

| `routes/components/agent.ts` | `parseAgentParams` 解析 body 中的 `userInfo` JSON 字符串，传递给 `invokeAgent / streamAgent` |

| `agent/subGraphs/applyGraph.ts` | 重构图结构，新增 `confirmNode`（interrupt 等待用户上传文件确认）和 `submitNode`（组装 AgentSubmitDto 回调 Java `/internal/agent/submit`） |
| `.env` | 新增 `JAVA_BACKEND_URL=http://localhost:8080` 和 `INTERNAL_SERVICE_KEY=id-ai-internal-secret-2024` |

### 1.3 前端（idfrontend）

## 四、已知限制与注意事项

| 事项 | 说明 |
|------|------|
| `grade` 字段类型 | `users.grade` 为 MySQL `YEAR` 类型，Java 映射为 `Integer`，Node 端 `enrollmentYear` 直接使用整型年份 |
| `full_name` 为 null | 若学生未绑定信息，`studentName` 为空字符串，Agent 仍可正常工作，但提交的申请 `student_name` 为空 |
| submitNode 重试 | 若用户格式错误，Agent 会提示正确 JSON 格式；需用户重新发送正确格式才能提交 |
| 环境变量 | `INTERNAL_SERVICE_KEY` 必须与 Java 的 `internal.service-key` 值一致（默认均为 `id-ai-internal-secret-2024`） |
| Node 18+ | `fetch` 为内置，无需额外安装 |

## 五、核心数据流

```
用户登录 → JWT 中含 userId
  ↓
前端发送 POST /api/agent/stream (multipart)
  ↓
AIAgentController.stream():
  - request.getAttribute("userId")   // AuthInterceptor 已解析
  - userMapper.selectById(userId)    // 查 users 表
  - buildUserInfoJson() → JSON 字符串
  ↓
AIAgentService.agentStream():
  - multipart 中新增 userInfo 字段
  - 转发给 Node Agent :3001/agent/stream
  ↓
Node Agent routes/agent.ts:
  - parseAgentParams() 解析 userInfo JSON
  - streamAgent({ ..., userInfo })
  ↓
LangGraph applyGraph:
  fetchPolicy → analyzeAndMatch → summarize
    → confirmRoute: 有匹配 → confirmNode (interrupt)
  ← 用户上传文件后确认
  → submitNode:
      fetch POST /internal/agent/submit
      Header: X-Internal-Service-Key
      Body: AgentSubmitDto
  ↓
InternalAgentController (Java):
  - 验证 Service Key
  - applicationService.submitFromAgent(dto)
  - INSERT score_applications + application_proofs
```
