## 三、测试用例

### T01 — 上下文追问（LLM 语义）

**前置条件**: 用户描述信息不完整，mainGraph 进入 `askForMoreNode`

**测试步骤**:
1. 打开 AI 智能申请对话框
2. 发送模糊消息："我参加了一个竞赛"
3. 等待 AI 回复

**预期结果**:
- AI 追问应包含"您已提到竞赛"类似的确认语，然后询问缺少的字段（如竞赛级别、名次）
- 追问不超过 60 字，语气亲切
- 不重复用户已说明的内容

**测试步骤**:
1. 发送："我参加了省级编程竞赛"

## 四、关键数据流验证

### 上下文超限流程

```
用户发送消息
  → agentService.streamAgent()
    → checkContextLimit(config) 
      → app.getState() → msgCount >= limit
        → yield { type: 'context_limit', data: { message: '...' } }
          → consumeSSE() → callbacks.onContextLimit()
            → aiContextLimitReached.value = true
              → 输入框禁用 + 警告横幅显示
```

### confirm-action 确认流程

```
Agent confirmNode interrupt
  → streamAgent() → yield { type: 'interrupt', data: { question, suggestions, requireFiles: true } }
    → onInterrupt() → push confirm-action 消息（acted: false）
      → 用户点击「确认提交」
        → handleAiConfirm()
          → msg.acted = true（气泡变「✅ 已操作」）
          → agentResumeStream(sessionId, '{"action":"confirm","proofFileIds":[...],"proofValues":[...]}')
            → applyGraph.submitNode 解析并调用 Java /internal/agent/submit
```

## 五、注意事项

1. **上下文计数** 基于 LangGraph state 中的 `messages` 数组长度（含所有 HumanMessage + AIMessage），每轮对话大约增加 2 条
2. **历史 confirm-action 气泡** 一旦 `acted = true`，即使刷新页面也仍然显示「✅ 已操作」（基于 Vue 响应式，刷新后重置）
3. **新对话** 只是前端重置 sessionId，LangGraph checkpoint 历史仍在 SQLite 中保留，不影响其他会话
4. **上下文默认值** 首次运行时从 `db/init.ts` 中的 `INSERT OR IGNORE` 插入，已有 DB 不会覆盖已有值；若需更新默认值，需要 Admin 手动修改
