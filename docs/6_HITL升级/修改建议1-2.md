# 修改建议 1-2：方案 A 详细设计（用户信息透传 + Agent 回调提交）

> 基于修改建议1.md，选择方案 A，增加用户信息透传设计。

---

## 一、整体数据流

```
前端（Vue）
  │  1. Authorization: Bearer <token>
  │  2. POST /api/agent/stream (message + file + sessionId)
  ▼
Java（idbackend）
  │  AuthInterceptor 解析 JWT → 取出 userId
  │  查 DB → 拼装 studentInfo
  │  multipart 转发给 Node Agent，新增 studentInfo 字段
  ▼
Node Agent（ID-AIDemo）
  │  parseAgentParams 解析 studentInfo，写入 state
  │  LangGraph 运行 classify → apply → applyGraph
  │  summarize 后进入 confirmNode → interrupt
  │  SSE 推送 interrupt 事件（包含匹配结果 + 告知用户上传证明材料）
  ▼
前端
  │  展示匹配结果 + 文件上传区 + 确认按钮
  │  用户上传证明文件 → 调普通文件上传接口 → 得到 fileId 列表
  │  用户点击确认 → resume，把 {confirm:true, proofFileIds, proofValues} JSON 字符串发回
  ▼
Node Agent（恢复）
  │  submitNode 解析 resume 数据
  │  用 state.studentInfo + resume 数据 + state 里的申请结果
  │  HTTP POST 回调 Java /internal/agent/submit
  │  SSE 推送 result 事件（"申请提交成功，编号 xxx"）
  ▼
Java（接收回调）
  │  校验 X-Internal-Service-Key
  │  写入 DB，返回 applicationId
  ▼
前端
  显示成功消息
```

// 在 AIAgentController 中新增
private String buildStudentInfoJson(HttpServletRequest httpRequest) {
    Integer userId = (Integer) httpRequest.getAttribute("userId");
    if (userId == null) return "{}";
    
    // 查用户信息（注入 UserProfileMapper 或 UserService）
    UserProfilePO profile = userProfileMapper.findByUserId(userId);
    if (profile == null) return "{}";
    
    Map<String, Object> info = new HashMap<>();
    info.put("studentId", profile.getStudentId());       // 学号
    info.put("studentName", profile.getFullName());      // 姓名
    info.put("major", profile.getMajor());               // 专业
    info.put("enrollmentYear", profile.getGrade());      // 入学年份
    return JSON.toJSONString(info);
}

### 2.4 Java：新增内部回调接口 `/internal/agent/submit`

Agent 在提交时会回调 Java。需要新增一个不走 JWT 验证的内部接口，用 **Service Secret** 做轻量鉴权。

```java
// 新建 InternalAgentController.java
@RestController
@RequestMapping("/internal/agent")
public class InternalAgentController {

    @Value("${internal.service-key}")
    private String serviceKey;

    private final ScoreApplicationService scoreApplicationService;

    @PostMapping("/submit")
    public ResponseEntity<Map<String, Object>> submit(
            @RequestHeader("X-Internal-Service-Key") String key,
            @RequestBody AgentSubmitDto dto) {

        if (!serviceKey.equals(key)) {
            return ResponseEntity.status(403).body(Map.of("code", 403, "msg", "forbidden"));
        }

        Long applicationId = scoreApplicationService.submitFromAgent(dto);
        return ResponseEntity.ok(Map.of("code", 200, "data", Map.of("applicationId", applicationId)));
    }
}
```

`AgentSubmitDto` 字段（Node Agent 传过来的 JSON）：
```java
public class AgentSubmitDto {
    private String studentId;
    private String studentName;
    private String major;
    private Integer enrollmentYear;
    private Integer templateId;       // 来自 applyGraph suggestions
    private Integer ruleId;
    private Double applyScore;
    private String remark;
    private List<ProofFileItem> proofItems;  // {fileId, proofValue, remark}
}
```

**Security Config** 里要把 `/internal/**` 加入白名单（不走 JWT filter），但只对内网 IP 开放，或简单用 Secret 头控制。

`application.properties` 新增：
```properties

### 2.8 Node Agent：applyGraph.ts 新增 confirmNode 和 submitNode

这是改动最多的地方。图结构从：
```
fetchPolicy → analyzeAndMatch → summarize → END
```
改为：
```
fetchPolicy → analyzeAndMatch → summarize → confirm(interrupt) → submit → END
                                                                ↘ cancel → END
```

**confirmNode**：interrupt 暂停，展示匹配结果，要求用户上传证明材料并确认：

```typescript
async function confirmNode(state: ApplyStateType): Promise<Partial<ApplyStateType>> {
  const suggestions = state.checkResults
    .map(r => { try { return JSON.parse(r) } catch { return null } })
    .filter(Boolean)

  if (suggestions.length === 0) return {}  // 无匹配直接结束

  const summary = suggestions.map((s: any) =>
    `• ${s.templateName} / ${s.ruleName}（预计 ${s.estimatedScore} 分）`
  ).join('\n')

  const question = [
    `已为您匹配到以下加分项：\n${summary}`,
    '',
    '请上传对应的证明材料，然后回复：',
    '```json',
    '{"confirm":true,"proofFileIds":[文件ID...],"proofValues":[对应分值...]}',
    '```',
    '或回复 cancel 取消。'
  ].join('\n')

  // interrupt 暂停，等待用户回复
  const userAnswer = interrupt(question)
  return {
    messages: [
      new AIMessage(question),
      new HumanMessage(String(userAnswer))
    ]
  }
}
```

**submitNode**：解析用户回复，调用 Java 内部接口提交：

```typescript
import fetch from 'node-fetch'  // 或 Node 18+ 原生 fetch

const JAVA_URL = process.env.JAVA_BACKEND_URL ?? 'http://localhost:8080'
const SERVICE_KEY = process.env.INTERNAL_SERVICE_KEY ?? ''

async function submitNode(state: ApplyStateType): Promise<Partial<ApplyStateType>> {
  const lastHuman = state.messages
    .filter(m => m instanceof HumanMessage)
    .at(-1)
  const answer = String(lastHuman?.content ?? '').trim()

  if (answer.toLowerCase() === 'cancel') {
    return { messages: [new AIMessage('已取消申请。')] }
  }

  // 解析用户回复的 JSON
  let proofFileIds: number[] = []
  let proofValues: number[] = []
  try {
    const parsed = JSON.parse(answer)
    if (!parsed.confirm) throw new Error('未确认')
    proofFileIds = parsed.proofFileIds ?? []
    proofValues = parsed.proofValues ?? []
  } catch {
    return { messages: [new AIMessage('格式不正确，请重新发送确认信息，或回复 cancel 取消。')] }
  }

  // 取第一个 suggestion（最优匹配）
  const suggestion = state.checkResults
    .map(r => { try { return JSON.parse(r) } catch { return null } })
    .filter(Boolean)[0]

  if (!suggestion || !state.studentInfo) {
    return { messages: [new AIMessage('申请数据不完整，无法提交。')] }
  }

  const submitBody = {
    studentId: state.studentInfo.studentId,
    studentName: state.studentInfo.studentName,
    major: state.studentInfo.major,
    enrollmentYear: state.studentInfo.enrollmentYear,
    templateId: suggestion.templateId,
    ruleId: suggestion.ruleId,
    applyScore: suggestion.estimatedScore,
    remark: `AI 智能申请 - ${suggestion.reason}`,
    proofItems: proofFileIds.map((id: number, i: number) => ({
      fileId: id,
      proofValue: proofValues[i] ?? 0,
      remark: ''
    }))
  }

  try {
    const resp = await fetch(`${JAVA_URL}/internal/agent/submit`, {
      method: 'POST',
      headers: {
        'Content-Type': 'application/json',
        'X-Internal-Service-Key': SERVICE_KEY
      },
      body: JSON.stringify(submitBody)
    })
    const data = await resp.json() as any
    if (data.code === 200) {
      return { messages: [new AIMessage(`✅ 申请已提交成功！申请编号：${data.data?.applicationId}`)] }
    }
    return { messages: [new AIMessage(`提交失败：${data.msg}`)] }
  } catch (e) {
    return { messages: [new AIMessage(`提交失败，请稍后重试或手动申请。`)] }
  }
}
```

**confirmRoute**：根据用户回复决定走 submit 还是 cancel：

```typescript
function confirmRoute(state: ApplyStateType): 'submit' | 'cancel' {
  const lastHuman = state.messages
    .filter(m => m instanceof HumanMessage)
    .at(-1)
  const answer = String(lastHuman?.content ?? '').trim().toLowerCase()
  return answer === 'cancel' ? 'cancel' : 'submit'
}
```

**图结构更新**：

```typescript
export const applySubgraph = new StateGraph(ApplyState)
  .addNode('fetchPolicy', fetchPolicyNode)
  .addNode('analyzeAndMatch', analyzeAndMatchNode)
  .addNode('summarize', summarizeNode)
  .addNode('confirm', confirmNode)          // ← 新增
  .addNode('submit', submitNode)            // ← 新增

  .addEdge(START, 'fetchPolicy')
  .addEdge('fetchPolicy', 'analyzeAndMatch')
  .addEdge('analyzeAndMatch', 'summarize')
  .addEdge('summarize', 'confirm')          // ← 改：原来是 summarize → END
  .addConditionalEdges('confirm', confirmRoute, {   // ← 新增

    cancel: END
  })
  .addEdge('submit', END)                   // ← 新增
  .compile()
```

## 六、关键注意事项

**① interrupt 在 consult 路径下不触发 confirm**

`confirmNode` 只在 `applyGraph` 里，`consultGraph` 走完直接 END，不会有文件上传这一步。逻辑是清晰隔离的。

**② summarize 无匹配时不进 confirm**

当 `summarize` 发现 `suggestions` 为空时，直接返回"未匹配"消息，后续 `confirmNode` 中检查无 suggestions 直接 END（不 interrupt），避免在无结果时还让用户上传文件。

**③ studentInfo 为 null 时（未登录/匿名）**

`submitNode` 检测到 `state.studentInfo` 为 null，返回"请先登录后再申请"，不发 HTTP 请求。

**④ proofItems 为空时**

`submitNode` 检测到 `proofFileIds` 为空数组，返回"请至少上传一个证明材料"，重新 interrupt 等待用户补充（或直接提示错误）。

**⑤ Internal Service Key 安全性**

`INTERNAL_SERVICE_KEY` 是一个随机 UUID，仅在内网使用，在生产环境配合网络隔离（Agent 和 Java 在同一内网）。不要把这个 key 暴露给前端。
