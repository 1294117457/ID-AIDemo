##### 1.schema测试验证

0424
主要发现的可优化点（15条有改进建议）：

```
	1.entity type 分类偏差：Python、Vue、React、Docker、签证、互联网+大赛等被标为 product，但它们更适合 topic 或 other 类型
	2.constraints 过度推断：如"头疼吃什么药"自动补充了"安全有效""常见非处方药"，"体检查什么"补充了"基础健康筛查"，这些原句未明确表述
	3.entity + constraints 冗余：如 GPA3.5 同时出现在 entities 和 constraints 中
	4.实体简称 vs 全称不一致：有时用"清华"有时用"清华大学"，需要统一规范
	5.复合意图处理不足：当一句话同时包含"对比"和"推荐"时，只取了一个
	这些发现对后续训练轻量提纯模型时的训练数据标注非常有指导价值。	
```

优化 schema 定义

```
1.Python,vue,docker标记为product，
	说明entity type定义不够清晰
2.constraints过度判断
	模型回复又过度的约束条件：安全有效、常见非处方药
	应给prompt强调只提取原文表达，不过度推断补充
3.复合意图的输入判断不充分
	intent_type只取主意图
```
