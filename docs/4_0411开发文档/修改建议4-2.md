## 架构方案对比

### 方案 A：纯代理（当前架构，增强版）

```
进入页面 → 前端请求 → Spring 代理 → Agent 查 SQLite → 返回文件列表
上传文件 → 前端请求 → Spring 代理 → Agent 解析+嵌入+入库 → 返回详细状态
```

- **数据唯一来源**：Agent 的 SQLite
- Spring 后端只做 HTTP 透传，不额外存储
- 每次进入页面都实时查 Agent

| 优点 | 缺点 |
|------|------|
| 架构简单，数据不重复 | Agent 离线时页面显示空白 |
| 始终展示最新状态 | 每次进入页面都要等 Agent 响应 |
| 不需要额外数据库表 | 上传过程中无"中间状态"可追踪 |

### 方案 B：后端额外存储上传记录（MySQL）

```
进入页面 → 前端请求 → Spring 查 MySQL → 返回文件列表（含状态字段）
上传文件 → Spring 先写 MySQL 记录(status=uploading)
         → 转发给 Agent
         → Agent 返回成功/失败
         → Spring 更新 MySQL 记录(status=success/failed, chunk_count=N)
```

- **数据双写**：MySQL 存上传记录+状态，Agent SQLite 存实际向量数据
- 页面数据来源：MySQL（快，不依赖 Agent 在线）

| 优点 | 缺点 |
|------|------|
| Agent 离线时仍能看到历史上传记录 | 数据两份，需保持同步 |
| 可追踪上传中间状态 | 如果 Agent 重启后 SQLite 被清空，MySQL 状态与实际不一致 |
| 页面加载更快（直接查 MySQL） | 多一张表 + 后端多一层逻辑 |

### 方案推荐：**方案 A（纯代理增强版）**

理由：
1. 知识库的消费者**只有 Agent**，数据存在 Agent 端是自然的
2. Agent 离线时知识库功能本身就不可用，MySQL 中存一份"历史记录"意义不大
3. 项目目前规模不大，避免引入同步复杂度
4. 只需增强 Agent 的响应信息 + 前端的状态展示即可满足需求

```typescript
if (!text.trim() || text === '（文件解析结果为空）') {

    code: 400,
    msg: '文件内容为空或无法解析',
    data: { fileName, status: 'parse_empty' }  // 新增状态码

}
```

catch 块返回更详细的错误：

```typescript
catch (err) {
  console.error('[knowledge/upload] 完整错误:', err)

  // 区分解析失败和嵌入失败
  const status = errMsg.includes('embedding') || errMsg.includes('API')
    ? 'embed_failed' : 'process_failed'
  res.json({
    code: 500,
    msg: `处理失败: ${errMsg}`,
    data: { fileName, status }
  })
}
```

### 3. 管理端前端（重点改动）

#### 3.1 API 层 `apiAIagent.ts` — 增强类型

```typescript
/** 上传结果 */
export interface UploadResult {
  fileName: string
  chunkCount: number
  textLength?: number
  status: 'success' | 'parse_empty' | 'embed_failed' | 'process_failed'
}

/** 知识库统计 */
export interface KnowledgeStats {
  totalFiles: number
  totalChunks: number
  files: KnowledgeFile[]
}

/** 上传知识库文件 */
export const uploadKnowledge = async (file: File): Promise<ApiResponse<UploadResult>> => {
  const formData = new FormData()
  formData.append('file', file)
  const response = await apiClient.post(`${apiBaseUrl}/api/knowledge/upload`, formData, {
    headers: { 'Content-Type': 'multipart/form-data' },
    timeout: 300000,   // 5分钟，大文件嵌入耗时长
  })
  return response.data
}

/** 获取知识库统计 */
export const getKnowledgeStats = async (): Promise<ApiResponse<KnowledgeStats>> => {
  const response = await apiClient.get(`${apiBaseUrl}/api/knowledge/stats`)
  return response.data
}
```

#### 3.3 KnowledgePanel 关键逻辑

**进入页面时**：
```typescript
onMounted(async () => {
  await loadStats()    // GET /api/knowledge/stats → 同时获取文件列表和统计
})
```

用 `/stats` 一次请求取回文件列表 + 统计数据，不需要分两个请求。

**上传后**：
```typescript
const handleUpload = async (e: Event) => {
  // ...
  uploading.value = true
  uploadResult.value = null
  try {
    const res = await uploadKnowledge(file)
    if (res.code === 200 && res.data) {
      uploadResult.value = {
        success: true,
        fileName: res.data.fileName,
        message: `${res.data.fileName} 上传成功，解析 ${res.data.textLength ?? '?'} 字 → ${res.data.chunkCount} 个向量分块`
      }
      ElMessage.success(uploadResult.value.message)
      await loadStats()  // 刷新列表和统计
    } else {
      uploadResult.value = {
        success: false,
        fileName: file.name,
        message: res.msg || '上传失败'
      }
      ElMessage.error(uploadResult.value.message)
    }
  } catch {
    uploadResult.value = {
      success: false,
      fileName: file.name,
      message: '上传失败，请检查 AI Agent 是否运行'
    }
    ElMessage.error(uploadResult.value.message)
  } finally {
    uploading.value = false
  }
}
```

**上传反馈区**（在表格下方展示最近一次上传结果）：
```html
<div v-if="uploadResult" class="mt-4">
  <el-alert
    :type="uploadResult.success ? 'success' : 'error'"
    :title="uploadResult.message"
    :closable="true"
    @close="uploadResult = null"
  />
</div>
```

## 数据流总结

```
┌── 进入 AI管理 > 知识库 Tab ──┐
│                               │
│  GET /api/knowledge/stats     │
│    → Spring 代理              │
│    → Agent GET /knowledge/stats
│    → 查 SQLite                │
│    → 返回 { totalFiles, totalChunks, files[] }
│    → 前端渲染统计 + 列表      │
│                               │
├── 上传文件 ──────────────────┤
│                               │
│  POST /api/knowledge/upload   │
│    → Spring multipart 代理    │
│    → Agent 接收文件           │
│    → MCP parse_document       │
│    → MCP ingest_document      │
│    → 返回 { fileName, chunkCount, textLength, status }
│    → 前端显示上传结果反馈     │
│    → 自动刷新 stats           │
│                               │
├── 删除文件 ──────────────────┤
│                               │
│  DELETE /api/knowledge/{file} │
│    → Spring 代理              │
│    → Agent 删除 SQLite 记录   │
│    → 前端自动刷新 stats       │
│                               │
└───────────────────────────────┘
```

全部数据来源为 Agent SQLite，后端不额外存储，前端每次操作后实时刷新。
