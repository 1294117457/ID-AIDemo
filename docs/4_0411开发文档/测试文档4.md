# 测试文档 4 — 知识库增强 + AI管理页面合并

---

## 一、本次修改概要

| # | 修改位置 | 改动 |
|---|---------|------|
| 1 | Agent `routes/knowledge.ts` | 上传响应增加 `textLength`/`status` + 详细日志 + 新增 `GET /knowledge/stats` |
| 2 | Agent `services/aiConfig.ts` | 缓存加 60s TTL，解决 MCP 子进程配置不同步 |
| 3 | Agent `index.ts` | 启动日志新增 `/knowledge/stats` |
| 4 | Backend `KnowledgeController.java` | 新增 `GET /api/knowledge/stats` 代理方法 |
| 5 | Backend `AIAgentService.java` | 新增 `getKnowledgeStats()` |
| 6 | Admin API `apiAIagent.ts` | `UploadResult` / `KnowledgeStats` 类型 + `getKnowledgeStats()` + 超时改 5min |
| 7 | Admin `views/ai-manage/index.vue` | 合并页面，`el-tabs` 双标签 |
| 8 | Admin `views/ai-manage/AiConfigPanel.vue` | 从旧 `ai-config/index.vue` 搬入 |
| 9 | Admin `views/ai-manage/KnowledgePanel.vue` | 从旧 `knowledge/index.vue` 搬入 + 统计卡片 + 上传反馈 |
| 10 | Admin `router/modules/ai-manage.ts` | 合并路由 `/home/ai-manage` |
| 11 | 删除旧文件 | `router/modules/ai-config.ts`、`knowledge.ts`、`views/ai-config/`、`views/knowledge/` |

## 二、测试环境准备

### 2.1 启动 Agent

```bash
cd LangChain升级文档/project/ID-AIDemo
npm run dev
```

**验证**：控制台应输出接口列表，包含新增的 `GET /knowledge/stats`。

### 2.2 启动 Spring Backend

```bash
cd LangChain升级文档/project/idbackend
mvn spring-boot:run
```

### 2.3 启动管理端前端

```bash
cd LangChain升级文档/project/idfrontend-admin
npm run dev
```

## 三、测试用例

### 3.1 菜单合并验证

| 编号 | 操作 | 预期结果 |
|------|------|---------|
| 3.1.1 | 登录管理端，查看左侧菜单 | 原来的"AI配置"和"AI知识库"两个菜单项消失，取而代之的是一个"AI管理"菜单项 |
| 3.1.2 | 点击"AI管理" | 进入页面，顶部有两个 Tab：「AI 配置」和「AI 知识库」，默认选中「AI 配置」 |
| 3.1.3 | 点击「AI 知识库」Tab | 切换到知识库面板，显示统计卡片 + 文件列表 |
| 3.1.4 | 点击「AI 配置」Tab | 切换回配置面板，表单数据保持不变 |

### 3.2 知识库统计卡片

| 编号 | 操作 | 预期结果 |
|------|------|---------|
| 3.2.1 | 进入「AI 知识库」Tab | 顶部显示三个统计卡片：知识库文件数、向量分块总数、Agent 状态 |
| 3.2.2 | Agent 在线时 | Agent 状态显示绿色「● 在线」 |
| 3.2.3 | 停止 Agent 后刷新页面 | Agent 状态显示红色「● 离线」，出现黄色警告提示 |
| 3.2.4 | 有知识库文件时 | 文件数和分块数大于 0，与表格中的数据一致 |

### 3.3 知识库上传增强

| 编号 | 操作 | 预期结果 |
|------|------|---------|
| 3.3.1 | 点击「上传文件」按钮 | 弹出文件选择器，支持 PDF/Word/Excel/Markdown/TXT |
| 3.3.2 | 选择一个有效文件上传 | 按钮文字变为"解析入库中..."；上传完成后表格下方出现绿色反馈条，显示"xxx 上传成功，解析 N 字 → M 个向量分块" |
| 3.3.3 | 上传成功后检查统计 | 文件数 +1，分块总数增加，文件列表新增一行 |
| 3.3.4 | 上传内容为空的文件 | 表格下方出现红色反馈条，提示"文件内容为空或无法解析" |
| 3.3.5 | Agent 离线时上传 | 出现红色反馈条，提示"上传失败，请检查 AI Agent 是否运行" |
| 3.3.6 | 点击反馈条的关闭按钮 | 反馈条消失 |

### 3.4 知识库删除

| 编号 | 操作 | 预期结果 |
|------|------|---------|
| 3.4.1 | 点击某文件行的「删除」按钮 | 弹出确认气泡 |
| 3.4.2 | 确认删除 | 文件从列表中消失，统计数字刷新（文件数 -1，分块数减少） |
| 3.4.3 | 删除后向 AI 提问该文件内容 | AI 无法回答（该知识已从向量库移除） |

### 3.5 AI 配置 Tab

| 编号 | 操作 | 预期结果 |
|------|------|---------|
| 3.5.1 | 进入「AI 配置」Tab | 显示当前配置：系统提示词、对话模型、向量模型、API Base URL、API Key（掩码） |
| 3.5.2 | 修改系统提示词并保存 | 提示"配置已保存，立即生效" |
| 3.5.3 | 保存后新开对话 | AI 回复风格符合新的系统提示词 |
| 3.5.4 | 修改向量模型名称 | 弹出警告对话框提示"向量索引将失效" |
| 3.5.5 | 点击「重置」 | 表单恢复为服务端当前值 |

### 3.6 Agent 端日志验证

| 编号 | 操作 | 预期结果 |
|------|------|---------|
| 3.6.1 | 上传一个文件后查看 Agent 控制台 | 输出 `[knowledge/upload] 解析完成: xxx, 文本长度=N` |
| 3.6.2 | 同上 | 输出 `[knowledge/upload] 入库完成: xxx, result=已入库：xxx，共 M 个分块` |
| 3.6.3 | 上传空文件或不支持格式 | 输出 `[knowledge/upload] 解析完成: xxx, 文本长度=0` 或完整错误信息 |

### 3.8 配置 TTL 缓存验证

| 编号 | 操作 | 预期结果 |
|------|------|---------|
| 3.8.1 | 在管理端修改 API Key 并保存 | 保存成功 |
| 3.8.2 | 立即在学生端发起对话 | 主进程 `invalidate()` 立即清缓存，新对话使用新 Key |
| 3.8.3 | 修改配置后等待 60 秒再上传知识库文件 | MCP 子进程 TTL 过期后读到新配置，嵌入使用新 Key |
