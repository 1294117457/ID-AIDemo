## 一、本次修改内容

### 1.1 Agent 端：过滤 classifyNode 的 JSON token

**文件**：`ID-AIDemo/src/agent/mainGraph.ts`

**改动**：`streamAgent()` 和 `streamResume()` 两个函数的 `streamEvents` 循环中，新增节点白名单过滤：

```typescript
const ANSWER_NODES = new Set(['answer', 'summarize'])

for await (const event of eventStream) {
  if (event.event === 'on_chat_model_stream') {
    const node = event.metadata?.langgraph_node
    if (node && !ANSWER_NODES.has(node)) continue  // 跳过非回答节点
    const token = event.data?.chunk?.content
    if (token) yield { type: 'token', data: { content: token } }
  }
}
```

**效果**：
- `classifyNode`（输出 `{"intent":"consult",...}` JSON）的 token 被过滤
- `applyGraph > analyzeAndMatchNode`（输出 suggestions JSON）的 token 被过滤
- 只有 `consultGraph > answer` 和 `applyGraph > summarize` 的自然语言输出会流式推送

### 1.2 前端：Markdown 渲染

**文件**（4 个）：
- `idfrontend/package.json` — 新增 `marked`、`dompurify`、`@types/dompurify`
- `idfrontend/src/views/ai-agent/index.vue` — 添加 `renderMarkdown()`，AI 气泡改用 `v-html`
- `idfrontend-admin/package.json` — 同上
- `idfrontend-admin/src/views/ai-agent/index.vue` — 同上

**改动详情**：

1. **script 新增**：
   - `import { marked } from 'marked'` + `import * as DOMPurify from 'dompurify'`
   - `renderMarkdown(text)` 函数：`marked.parse()` → `DOMPurify.sanitize()` → 安全 HTML

2. **template 改动**（2 处）：
   - AI 消息气泡：`{{ msg.content }}` → `v-html="renderMarkdown(msg.content)"`
   - interrupt 气泡：同上

3. **style 新增**：
   - 非 scoped 的 `.markdown-body` 样式，覆盖 `p`、`ul`/`ol`/`li`、`strong`、`blockquote`、`code`/`pre`、`table`、`h1`-`h3`、`hr`、`a` 等元素

## 六、已知限制

1. **流式 Markdown 中间状态**：流式输出时，`marked.parse()` 可能对半截的 Markdown 产生临时的不完整渲染（如粗体标记只出现了左半 `**`），这在最终 token 到达后会自动修复。
2. **代码高亮**：当前 Markdown 的 `<pre><code>` 只有深色背景，没有语法高亮。如需高亮可后续引入 `highlight.js`，但当前场景（政策咨询）很少涉及代码块，暂不需要。
3. **`.markdown-body` 样式全局性**：CSS 使用非 scoped 的 `<style>` 块，`.markdown-body` 类名会全局生效。命名足够特殊，不太可能和其他组件冲突。
