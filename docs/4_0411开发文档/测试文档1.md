## 二、测试前提条件

### 环境要求

- Node.js >= 18
- ID-AIDemo 已安装最新依赖：`cd project/ID-AIDemo && npm install`
- `.env` 配置正确（QWEN3_API_KEY / QWEN_BASE_URL 等）
- 知识库至少有一个已入库文件（通过 `/knowledge/upload` 或 seed）
- idbackend 正常运行于 `:8080`
- idfrontend / idfrontend-admin 正常运行

### 启动顺序

```
1. cd project/ID-AIDemo && npm run dev     → http://localhost:3001
2. cd project/idbackend && mvn spring-boot:run  → http://localhost:8080
3. cd project/idfrontend && npm run dev    → http://localhost:5173
4. cd project/idfrontend-admin && npm run dev → http://localhost:5174
```

#### TC-1.1 非流式咨询（/agent/chat）

**请求**：
```bash
curl -X POST http://localhost:3001/agent/chat \
  -F "message=挑战杯能加多少分" \
  -F "sessionId=test_consult_001"
```

**预期响应**：
```json
{
  "code": 200,
  "msg": "成功",
  "data": {
    "interrupted": false,
```

**验证点**：

## 四、异常场景测试

| 编号 | 场景 | 操作 | 预期 |
|------|------|------|------|
| ERR-1 | 空消息 | POST /agent/chat body 无 message | `{ code: 400, msg: "请输入文字或上传文件" }` |

## 五、完整链路验证（端到端）

### E2E-1：咨询完整链路

```
idfrontend
  → agentStreamChat("挑战杯能加多少分", sessionId)
  → fetch POST /api/agent/stream（带 Bearer token）
    → idbackend AIAgentController.stream()
      → AIAgentService.agentStream()
        → POST http://localhost:3001/agent/stream（multipart/form-data）
          → routes/agent.ts → streamAgent()
            → mainGraph: classify → consult
              → consultGraph: retrieveNode(MCP search_knowledge) → answerNode(LLM)
          → SSE 逐 token 返回
        ← SSE 透传回 idbackend
      ← SSE 透传给前端
    ← SseEmitter → fetch ReadableStream
  ← 逐 token 追加到消息气泡
```

### E2E-2：申请加分完整链路（带 interrupt）

```
idfrontend
  → agentStreamChat("我想申请加分", sessionId)
  → ... → classify → insufficient → ask → interrupt
  ← 前端收到 interrupt 事件，显示补充信息气泡

用户补充 "2024年挑战杯，金奖，负责人"
  → agentResumeStream(sessionId, supplement)
  → ... → resume → classify → apply → applyGraph
    → fetchPolicy(MCP) → analyzeAndMatch(LLM) → summarize
  ← 前端逐 token 显示匹配结果

用户选择一项 suggestion
  → POST /api/ai/analyze/generate（走旧路径）
  ← 返回 remark, applyScore 等
  → 前端预填申请表单
```

## 六、已知限制和注意事项

| 项目 | 说明 |
|------|------|
| MemorySaver | 纯内存存储，Agent 重启后所有会话丢失。符合需求（对话不需持久化） |
| MCP stdio | MCP Server 作为 Agent 的子进程运行。首次调用有约 1-2s 启动延迟 |
| Windows 兼容性 | MCP stdio transport 在 Windows 上可能有编码问题，如遇到需改用 HTTP transport |
| sessionId | 前端每次打开对话窗口自动生成新的 sessionId，不复用旧线程 |
| 旧接口并存 | `/chat/*`、`/analyze/*` 旧接口完全保留不动，新旧 sessionId 不互通 |
| 模板加载 | 通过后端代理时模板从 MySQL 自动加载；直接调 Agent 时走 MySQL 或传入 templates 参数 |
