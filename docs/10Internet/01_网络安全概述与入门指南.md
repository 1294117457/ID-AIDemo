# 网络安全概述与入门指南

## 一、什么是网络安全

网络安全的核心目标是保护信息系统的 **CIA 三要素**：

| 要素 | 含义 | 被破坏的例子 |
|------|------|-------------|
| **机密性（Confidentiality）** | 数据只有授权的人能看 | 用户密码泄露、数据库被拖 |
| **完整性（Integrity）** | 数据不能被篡改 | 网页被挂马、订单金额被改 |
| **可用性（Availability）** | 系统要能正常运行 | DDoS 攻击打瘫服务器 |

> 分析任何漏洞时，都可以问自己：**这个漏洞破坏了 CIA 中的哪个？**

## 二、黑客的分类

| 类型 | 做什么 | 合法性 |
|------|--------|--------|
| **白帽（White Hat）** | 在授权范围内发现漏洞，帮助企业修复安全问题 | 合法，有报酬 |
| **灰帽（Grey Hat）** | 未经授权发现漏洞，但不恶意利用，可能会通知厂商 | 法律灰色地带 |
| **黑帽（Black Hat）** | 未经授权入侵系统，窃取数据、勒索、破坏 | 违法犯罪 |

技术本身是一样的，**区别在于动机和是否有授权**。

## 三、安全领域的主要方向

| 方向 | 做什么 | 说明 |
|------|--------|------|
| **Web 渗透测试** | 对 Web 应用做安全测试，找漏洞 | 最适合有 Web 开发经验的人 |
| **代码审计** | 阅读源代码，找安全缺陷 | 需要能读懂代码 |
| **红队（Red Team）** | 模拟真实攻击者，全方位测试企业安全 | 进阶方向，需要更广的技术栈 |
| **蓝队（Blue Team）** | 防御方，做监控、应急响应、溯源分析 | 偏运维方向 |
| **安全开发（DevSecOps）** | 把安全融入开发流程，做安全工具、安全架构 | 结合开发能力 |
| **漏洞研究** | 研究 0day，挖浏览器/操作系统级别的漏洞 | 门槛高，需要逆向和二进制基础 |
| **智能合约审计** | 审计 Solidity 合约的安全性 | 适合有 Web3 经验的人 |
| **AI 安全** | LLM 安全、Agent 安全、对抗样本 | 新兴方向 |

## 四、核心安全概念

### 信任边界

系统中哪些数据是可信的，哪些不可信。**所有来自用户的输入都不可信** —— 这是安全的第一原则。前端校验不算数，必须在后端校验。

### 攻击面

一个系统暴露给外部的所有可交互点。每多开一个端口、每多一个接口、每多引入一个依赖，攻击面就扩大了。

### 最小权限原则

任何用户、程序、进程，只应该拥有完成其任务所需的最小权限。数据库连接不该用 root，API 接口该限权限就限权限。

### 纵深防御

不依赖单一防御手段。即使 WAF 被绕过，后端还有参数校验；即使 SQL 注入成功，数据库账户权限有限，拿不到敏感表。层层设防。

### 漏洞的生命周期

1. 漏洞被引入（开发者写了有问题的代码）
2. 漏洞被发现（白帽/黑帽/安全研究员发现了它）
3. 漏洞被披露（CVE 编号、安全公告）
4. 补丁发布
5. 用户打补丁

**0day** = 漏洞已被发现但还没有补丁的阶段，最危险。

### CVE 和 CVSS

- **CVE**：公共漏洞编号，比如 CVE-2021-44228 就是 Log4Shell
- **CVSS**：漏洞严重程度评分，0-10 分，10 分最严重

## 五、开发者视角 vs 攻击者视角

**开发者思维**：我怎么让功能跑起来？

**攻击者思维**：这个功能在什么情况下会出问题？

以登录接口为例：

- 开发者想的：用户输入账号密码，验证通过就返回 token
- 攻击者想的：
  - 密码错误有次数限制吗？没有的话可以爆破
  - 返回的错误信息区分"用户不存在"和"密码错误"吗？可以枚举用户名
  - Token 是怎么生成的？JWT 的密钥能伪造吗？
  - 登录接口有没有验证码？没有可以撞库
  - 忘记密码的流程安全吗？重置链接有过期时间吗？

> **白帽 = 用攻击者的思维审视系统，在授权范围内，最终目的是帮助修复。**

## 六、攻击链（Kill Chain）

真实的攻击不是单一漏洞，而是多个漏洞串联：

1. 信息收集发现目标的测试环境子域名（配置问题）
2. 测试环境上有未授权的 API 接口（认证缺陷）
3. 通过接口拿到内部员工邮箱列表（数据泄露）
4. 用邮箱发钓鱼邮件（社会工程学）
5. 有人点了，拿到内网 VPN 凭据（人的漏洞）
6. 进入内网发现 Redis 没设密码（配置漏洞）
7. 通过 Redis 写入计划任务拿到服务器权限（漏洞利用）

> 单独看每一步可能都不致命，但串起来就是一次完整的入侵。

## 七、法律红线

中国《网络安全法》《刑法》第285、286条对未经授权入侵计算机信息系统有明确的刑事处罚规定。

**合法行为：**
- 测试自己的系统
- 打靶场（DVWA、Hack The Box、TryHackMe）
- 参加 CTF 比赛
- 通过 SRC（安全应急响应中心）合法报漏洞

**违法行为：**
- 未经授权扫描/测试别人的服务器（哪怕"没有恶意"）
