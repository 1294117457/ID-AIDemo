### 今日任务

### 今日积累
- 联合注入的标准步骤（背下来）：
  1. `ORDER BY 1--`, `ORDER BY 2--` ... 找列数
  2. `UNION SELECT NULL,NULL,...--` 确定回显位

  5. `UNION SELECT column_name,NULL FROM information_schema.columns WHERE table_name='xxx'--`
  6. `UNION SELECT username,password FROM xxx--`

## Day 19 — CSRF 原理与实战

### 今日任务
- [ ] 理解 CSRF 的本质：利用浏览器自动携带 Cookie 的机制，让用户在不知情的情况下发起请求
- [ ] 理解 CSRF 的前提条件：用户已登录 + 浏览器自动发送 Cookie + 目标接口没有额外验证
- [ ] 完成 PortSwigger Lab：CSRF vulnerability with no defenses
- [ ] 在 DVWA CSRF 模块（Low 难度）练习修改密码的 CSRF 攻击

### 今日积累
- CSRF 攻击的标准流程：
  1. 用户登录了目标网站 A（浏览器保存了 Cookie）
  2. 攻击者构造恶意页面 B，页面中包含向网站 A 发请求的表单
  3. 用户在登录状态下访问页面 B
  4. 浏览器自动带上 A 的 Cookie 发送请求，操作成功
- CSRF 和 XSS 的区别：XSS 是在目标网站上执行脚本，CSRF 是从外部网站发起请求

- `SameSite=Lax`：导航级请求允许，Ajax 请求不允许（现代浏览器默认值）
- 前后端分离项目（用 JWT + Authorization Header）天然不受 CSRF 影响

### 今日笔记
对比 XSS 和 CSRF：攻击方式、前提条件、危害、防御手段。
