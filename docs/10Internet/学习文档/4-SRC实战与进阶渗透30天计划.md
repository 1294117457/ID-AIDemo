# 第十三周：SRC 资产测绘与信息收集

## Day 91 — SRC 目标选择与规则研究

### 今日任务
- [ ] 在补天平台浏览所有公开 SRC 项目，了解范围和规则
- [ ] 选择 3-5 个目标 SRC（建议先选范围大、资产多的厂商）
- [ ] 仔细阅读每个 SRC 的规则：哪些域名在范围内、哪些行为被禁止、漏洞评级标准
- [ ] 了解"公益 SRC"和"企业 SRC"的区别

### 今日积累
- ⭐ SRC 规则是红线，必须严格遵守
- 选择目标的原则：资产多（攻击面大）、更新频繁（新功能容易有漏洞）、安全成熟度中等
- 避免选顶级安全团队的目标（如 Google），初期选中小厂更容易出成果

## Day 92 — 资产发现：子域名收集

### 今日任务
- [ ] 对选定的 SRC 目标做全面的子域名收集
- [ ] 使用多种方式交叉验证：
  - 工具：`subfinder`、`amass`、`OneForAll`
  - 搜索引擎：`site:target.com`
  - 证书透明度：crt.sh
  - DNS 暴力枚举
- [ ] 对收集到的子域名做存活检测：`httpx`
- [ ] 整理资产清单

### 今日积累
- 子域名数量直接决定攻击面大小
- 多工具交叉使用可以发现更多子域名
- `httpx` 可以批量检测域名存活状态和返回标题、状态码、技术栈

### 今日练习
输出一份子域名资产清单，至少包含：域名、IP、状态码、标题、技术栈。

## Day 93 — 资产发现：端口扫描 + 指纹识别

### 今日任务
- [ ] 对目标 IP 做全端口扫描：`nmap -p- --min-rate 5000 target`
- [ ] 对开放端口做服务版本识别：`nmap -sV -sC -p 端口列表 target`
- [ ] 使用指纹识别工具：`whatweb`、`wappalyzer`（浏览器插件）
- [ ] 学习 `masscan`：比 nmap 更快的大规模端口扫描

### 今日积累
- 资产测绘的完整链：子域名 → IP → 端口 → 服务 → 技术栈 → 已知漏洞
- `masscan` 扫全网 65535 端口只需几分钟，但需要控制速率
- 非标准端口上的 Web 服务经常被忽视，是挖洞的好目标

---

## Day 94 — 资产发现：JS 分析 + 接口发现

### 今日任务
- [ ] 学习从 JS 文件中提取 API 接口路径：`LinkFinder`、`JSFinder`
- [ ] 学习从 JS 中发现敏感信息：AK/SK、API Key、内部域名、调试接口
- [ ] 使用 `gau`（Get All URLs）收集目标的历史 URL
- [ ] 使用 `waybackurls` 从 Wayback Machine 收集历史页面

### 今日积累
- ⭐ JS 文件是信息泄露的重灾区，前端打包后的 JS 经常包含后端接口路径和密钥
- 历史 URL 中可能包含已经下线但未删除的旧接口
- `gau` + `waybackurls` + `LinkFinder` 组合使用效果最好

### 今日练习
对目标网站的所有 JS 文件做分析，提取接口和敏感信息。

## Day 95 — 目录扫描 + 敏感文件发现

### 今日任务
- [ ] 用 `ffuf` / `dirsearch` 对目标做目录和文件扫描
- [ ] 关注高价值文件：`.git`、`.env`、`.svn`、`backup.zip`、`web.config`、`phpinfo.php`
- [ ] 学习 `ffuf` 的高级用法：FUZZ 参数、过滤响应大小/状态码、多位置 FUZZ
- [ ] 整理一份自己的扫描字典（合并多个常见字典）

### 今日积累
- `.git` 泄露利用工具：`GitHack`、`git-dumper` — 可以还原完整源码
- `.env` 文件经常包含数据库密码、API Key 等敏感信息
- `ffuf` 高级用法：`ffuf -u URL/FUZZ -w wordlist -mc 200 -fs 1234`（按状态码和大小过滤）

# 第十四周：SRC 漏洞挖掘实战

## Day 98 — SRC 挖洞：信息泄露类

### 今日任务
- [ ] 专注挖信息泄露类漏洞（最容易出成果的类型）：
  - Actuator 端点暴露
  - Swagger/API 文档未授权访问
  - `.git` / `.env` / `phpinfo` 泄露
  - 错误页面泄露堆栈信息
  - JS 中的 AK/SK / 内部 IP
- [ ] 对你的 SRC 目标逐个检查

### 今日积累
- 信息泄露是 SRC 新手最容易提交的漏洞类型
- 虽然评级通常是低危/中危，但积累漏洞数量和报告经验很重要

---

## Day 99 — SRC 挖洞：越权类

### 今日任务
- [ ] 注册目标网站的普通账号，测试所有接口的权限控制
- [ ] 测试水平越权：修改请求中的 ID 参数，看能否访问其他用户的数据
- [ ] 测试垂直越权：用普通用户的 token 访问管理接口
- [ ] 使用 Burp 的 `Autorize` 插件辅助越权检测

### 今日积累
- ⭐ 越权是 SRC 中产出最高的漏洞类型
- Autorize 插件：自动对比不同权限用户的请求，高亮未做权限校验的接口
- 测试思路：注册两个账号，用 A 的 token 请求 B 的资源

---

## Day 100 — SRC 挖洞：逻辑漏洞

### 今日任务
- [ ] 测试业务逻辑漏洞：
  - 支付金额篡改（改为 0.01 或负数）
  - 优惠券/积分/兑换码的重复使用
  - 短信验证码爆破（4-6 位数字穷举）
  - 密码重置流程缺陷
  - 接口顺序绕过（跳过验证步骤）
- [ ] 关注新上线的功能（更容易有漏洞）

### 今日积累
- 逻辑漏洞需要理解业务流程，自动化工具扫不出来
- 新功能上线后 1-2 周是挖洞的黄金期
- 移动端 APP 的接口比 Web 端更容易有越权和逻辑漏洞（很多开发认为 APP 接口不会被直接调用）

---

## Day 101 — SRC 挖洞：注入类 + SSRF

### 今日任务
- [ ] 测试 SQL 注入：对所有参数用 sqlmap 做基础检测
- [ ] 测试 SSRF：对所有接受 URL 参数的接口测试
- [ ] 测试 XSS：对所有有输出的输入点测试
- [ ] 使用 Burp 的被动扫描辅助发现

### 今日积累
- 现代 Web 应用经过框架保护后，直接的 SQL 注入和 XSS 越来越少
- 但框架的保护经常有遗漏：搜索功能、导出功能、日志查看功能
- SSRF 在现代微服务架构中反而更常见

## Day 107 — 密码攻击专题

### 今日任务
- [ ] 学习密码哈希类型识别：MD5、SHA1、SHA256、bcrypt、NTLM
- [ ] 学习 `hashcat` 基本用法：字典攻击、规则攻击、掩码攻击
- [ ] 学习 `John the Ripper` 基本用法
- [ ] 学习密码喷射（Password Spraying）：用少量常见密码尝试大量用户

### 今日积累
- hashcat 基本用法：`hashcat -m 哈希类型 hashfile wordlist`
- 常见哈希类型编号：0=MD5、100=SHA1、1000=NTLM、3200=bcrypt
- 密码喷射可以绕过账号锁定策略（每个账号只尝试几次）

---

## Day 108 — WAF 绕过技术

### 今日任务
- [ ] 学习常见 WAF 的工作原理：正则匹配、语义分析、行为分析
- [ ] 学习 WAF 绕过技术：
  - 编码绕过：URL 编码、Unicode、十六进制
  - 分块传输（chunked encoding）
  - HTTP 参数污染（HPP）
  - Content-Type 切换
  - 大小写和注释混淆
- [ ] 了解常见 WAF：云 WAF（阿里云盾、CloudFlare）、硬件 WAF

### 今日积累
- WAF 绕过是实战渗透中经常遇到的问题
- 分块传输绕过：`Transfer-Encoding: chunked` 将 payload 分成多个 chunk 发送
- HPP：同一参数传两次，WAF 检查第一个，后端用第二个
