# 白帽学习路线与技术栈规划

## 一、安全领域的两条大路线

### 路线 A：Web / 应用层安全（推荐优先）

攻击和防御的对象是 Web 应用、API、业务逻辑、云服务。

**需要的技术栈：**

| 层次 | 内容 |
|------|------|
| HTTP 协议 | 请求响应、Header、Cookie、缓存、重定向 |
| 前端 | HTML/JS/DOM、同源策略、XSS 原理 |
| 后端 | Java/Node/Python 的 Web 框架，ORM，认证授权机制 |
| 数据库 | SQL 语法，数据库权限模型 |
| Python | 写漏洞验证脚本、自动化工具、爬虫 |
| Linux | 命令行操作、权限、服务、日志 |
| 网络基础 | TCP/IP、DNS、端口、代理 |
| 工具 | Burp Suite、Nmap、sqlmap、dirsearch |

**不需要 C 和汇编。** 核心是理解应用层的协议和业务逻辑。

### 路线 B：二进制 / 底层安全

攻击和防御的对象是操作系统、浏览器引擎、固件、驱动程序。

**需要的技术栈：**

| 层次 | 内容 |
|------|------|
| C 语言 | 指针、内存管理、堆栈结构 |
| 汇编 | x86/ARM 指令集，能读懂反汇编输出 |
| 操作系统 | 进程、内存分页、系统调用、内核 |
| 逆向工程 | IDA Pro、Ghidra、动态调试 |
| 漏洞类型 | 缓冲区溢出、堆溢出、UAF、ROP 链 |

门槛高、周期长，通常需要 1-2 年才能入门产出成果。

> **类比：** Web 安全像全科医生，二进制安全像外科医生。不需要先学解剖学才能当全科医生。

## 二、推荐学习路线（分阶段）

### 第一阶段：工具和环境（1-2 周）

目标：能动手操作。

- 装 **Kali Linux**（虚拟机，VMware 或 VirtualBox），渗透测试专用系统
- 学会基本 **Linux 命令**：cd、ls、cat、grep、chmod、ps、netstat、curl、wget
- 装好 **Burp Suite Community**（抓包改包的核心工具）
- 学会用 Burp 代理浏览器流量，拦截和修改请求

### 第二阶段：Web 漏洞原理 + 靶场（1-2 个月）

按漏洞类型逐个学习，每学一个在靶场里打一遍：

| 顺序 | 漏洞类型 | 学习理由 |
|------|---------|---------|
| 1 | SQL 注入 | 最经典，原理最容易理解 |
| 2 | XSS | 做前端的人理解 DOM 操作 |
| 3 | CSRF | 理解浏览器的信任模型 |
| 4 | 文件上传 | 直观，效果明显 |
| 5 | SSRF | 理解服务端请求伪造 |
| 6 | 越权 | 业务逻辑漏洞入门 |
| 7 | 反序列化 | 用 Java 的人必须关注 |

### 第三阶段：Python 安全脚本（同步进行）

- 用 `requests` 库写 HTTP 请求
- 写简单的目录扫描器
- 写简单的端口扫描器
- 写漏洞验证的 PoC 脚本（Proof of Concept）
- 学会用 sqlmap、nmap 等工具的 Python API

> 目标不是写出完美工具，而是理解工具背后在做什么。

### 第四阶段：网络基础（穿插学习）

不需要专门花几个月学网络，遇到什么补什么：

- 打靶场遇到端口扫描 → 学 TCP 三次握手、端口状态
- 遇到 DNS 相关漏洞 → 学 DNS 解析流程
- 遇到内网渗透 → 学子网、路由、ARP
- 遇到抓包分析 → 学 Wireshark 基本操作

> **需要时学，立刻实践**，比从头到尾看一本网络教材高效得多。

### 第五阶段：完整渗透测试 + CTF（3-6 个月后）

- **Hack The Box / TryHackMe**：打完整靶机，从信息收集到拿到 root 权限
- **参加 CTF 比赛**：Web 方向题目，练实战思维
- **挖 SRC**：对国内厂商（补天平台、各大厂 SRC）合法挖漏洞

完整渗透测试流程：
1. 信息收集（子域名、端口、指纹识别）
2. 漏洞扫描与发现
3. 漏洞利用与验证
4. 权限提升（提权）
5. 横向移动
6. 编写渗透报告

### 第六阶段：差异化方向（6 个月后）

基础稳固后，选一个方向深入：

- **AI 安全**：LLM 应用渗透、Prompt Injection、Agent 安全审计
- **Web3 安全**：智能合约审计、DeFi 协议安全
- **代码审计**：Java/TypeScript 项目的白盒安全审计

---

## 三、C 和汇编什么时候需要学

**当且仅当决定往以下方向深入时：**

- 浏览器漏洞挖掘（V8 引擎、WebKit）
- 操作系统内核漏洞
- IoT / 嵌入式设备安全
- 恶意软件分析
- 逆向工程

这是安全领域的"硬核方向"，不是所有白帽都需要走这条路。很多年薪很高的安全工程师一辈子都在 Web 层和应用层工作。

---

## 四、安全方向类比

| 安全方向 | 类比 | 需要的深度 |
|---------|------|-----------|
| Web 渗透测试 | 全科医生 | 了解常见病，快速诊断治疗 |
| 代码审计 | 内科医生 | 看得懂体检报告（源代码），找到隐藏的病 |
| 二进制安全 | 外科医生 | 需要理解人体最底层的结构（CPU、内存） |
| AI/Web3 安全 | 新兴学科专家 | 领域新，经验少，但需求爆发中 |

---

## 五、总结

> **推荐路线：Linux 基础 + Burp Suite → Web 漏洞靶场 → Python 脚本 → 完整渗透实战 → AI/Web3 安全方向**
